# BookBreaker · 拆书破局技能

> 一本书的价值，不在于读完，在于用上。

## 技能目标

将一本书按主题智能拆解成多集播客内容稿。
每集输出充实的纯内容稿，直接喂给 NotebookLM，让它自动生成约 20 分钟的双人播客音频。

---

## 执行流程

### 第一步：读懂这本书 + 内容密度评估

拿到书之后，先完整读一遍，输出以下评估结果：

**① 核心主张**：这本书最想说的一句话是什么

**② 内容密度判断**：
- 高密度：每章都有新观点、新案例，信息量大 → 适合4-5集
- 中密度：核心观点清晰，但部分章节有重复 → 适合3集
- 低密度：核心观点就那几条，大量篇幅是重复举例 → 适合2集，诚实告知用户

**③ 主题板块梳理**：全书可以归纳为哪几个大主题

**⚠️ 重要：** 必须如实评估内容密度，不因为书厚就强行多拆几集。宁可2集内容充实，不要5集注水稀释。

---

### 第二步：规划分集方案

根据内容密度评估结果，给出分集建议：

| 内容密度 | 建议集数 |
|----------|----------|
| 低密度 | 2集 |
| 中密度 | 3集 |
| 高密度 | 4-5集 |

**分集原则：**
- 每集围绕一个完整的核心主题，有独立的起承转合
- 集与集之间内容严格不重复
- 每集覆盖书中2-4个相关章节或主题板块
- 集数宁少勿多

**输出分集大纲，格式如下：**

```
《书名》分集规划

内容密度评估：[高/中/低]，建议共X集，原因：[一句话说明]

第一集：[集标题]
- 覆盖章节/主题：[具体说明]
- 核心问题：[这集要回答的核心问题]
- 不包含：[明确说明哪些内容留给后面的集]

第二集：[集标题]
- 覆盖章节/主题：[具体说明]
- 核心问题：[这集要回答的核心问题]
- 不包含：[明确说明哪些内容留给后面的集]

（以此类推）
```

**等待用户确认分集方案。用户确认后，此方案锁定，后续生成每集内容时严格按照边界执行，不随意扩展或修改。**

---

### 第三步：生成"已用内容追踪表"

用户确认分集方案后，在开始写第一集之前，先建立一个追踪表：

```
《书名》已用内容追踪表

【第一集】[标题] — 状态：待写
核心案例：（写完后填入）
核心观点：（写完后填入）

【第二集】[标题] — 状态：待写
核心案例：（写完后填入）
核心观点：（写完后填入）

（以此类推）
```

**每集写完后，立即更新追踪表，记录本集用过的核心案例和观点。下一集开始前，必须先读追踪表，确保不重复使用已用过的案例和观点。**

---

### 第四步：逐集生成内容稿

**每集生成前的检查清单：**
- [ ] 已读本集的分集边界说明
- [ ] 已读已用内容追踪表，确认本集不会重复已用内容
- [ ] 明确本集核心问题

**每集内容稿的结构：**

```
# 《书名》第X集：[集标题]
# （共X集）

## 本集聚焦
[用2-3句话说清楚：这集讲什么、为什么重要、听完能得到什么]

[正文——以连续段落展开，每个主题板块之间用简短小标题分隔]

[板块一标题]

[大段连续文字，完整展开这个板块的内容：核心观点、背后逻辑、
具体案例、数据支撑、反例对比、作者原话、延伸分析……
段落之间自然衔接，像一篇深度文章]

[板块二标题]

[同上]

[板块三标题]

[同上]

## 本集核心结论

[用连续段落总结这集最重要的3-5个观点，不用列表，用文字自然收尾]

## 留给你思考的问题

[提出2-3个值得听众深入思考的问题，引发共鸣]
```

---

### 第五步：内容稿写作标准

**字数要求：**
- 目标区间：15000-22000字
- **以内容完整为准，不强制字数下限**
- 如果某集的内容本身只支撑15000字，就写15000字，不注水
- 如果内容丰富，写到22000字也没问题
- **宁可15000字的高质量，不要22000字的注水**

**内容深度要求：**

✅ 应该包含的：
- 书中的核心论点，完整展开，不只列标题
- 书中的具体案例、故事、数据，完整呈现细节
- 作者的推理过程：为什么这样说，依据是什么
- 不同观点之间的对比和辩证
- 方法论的具体步骤，不只是原则
- 读者可以直接应用的行动建议
- 帮助理解核心观点的背景知识

❌ 绝对不能出现的：
- 章节标题堆砌，没有内容展开
- 一句话带过重要观点
- 过度概括，丢失细节
- 重复使用已在前几集出现过的案例
- 为凑字数而重复表达同一观点

**格式要求（影响NotebookLM生成质量）：**
- **以连续段落为主**，这是最重要的格式要求
- 小标题只用于分隔不同板块，不要过多
- **禁止大量使用bullet point和编号列表**——列表格式会让NotebookLM生成"汇报式"对话，而不是自然交流
- 段落之间自然衔接，像一篇深度文章，而不是PPT大纲

---

### 第六步：每集完成后的操作

每集写完后，立即执行以下两件事：

**① 更新追踪表**
把本集用过的核心案例和主要观点，填入追踪表对应位置。

**② 告知用户**

```
第X集「[集标题]」完成。
字数：约XX字
本集覆盖：[简要说明本集讲了什么]
已用核心案例：[列出本集用过的主要案例，防止后续集重复]

NotebookLM使用提示（复制粘贴到输入稿开头）：
---
这是《书名》播客系列的第X集，共X集，本集主题是「[集标题]」。
请基于以下内容，生成一段深度、自然的双人对话播客。
对话要求：深入探讨内容的核心观点，用具体案例和故事展开，
避免走马观花式的列举，要有来回的讨论、追问和共鸣，
让听众感觉像在听两个真正读过这本书的人在认真聊天。
---
[内容稿正文粘贴在此处]
```

---

### 第七步：全部完成后的总结

所有集数写完后，输出最终使用说明：

```
📌《书名》拆书完成

共X集：
- 第一集：[标题]，约XX字
- 第二集：[标题]，约XX字
（以此类推）

NotebookLM操作步骤：
1. 在 NotebookLM 新建一个项目，命名为「[书名]-拆书破局」
2. 将第一集内容稿（含开头的提示语）粘贴进去，生成音频
3. 在同一个项目里，继续粘贴第二集，生成音频
4. 重复直到全部集数完成
5. 同一个项目内，两位主播的声音全程保持一致

注意：每次粘贴新一集之前，不要新建项目，始终在同一个项目里操作。
```

---

## 特殊情况处理

**书的PDF文字提取有乱码或缺失：**
- 告知用户哪些部分无法读取，影响范围有多大
- 基于可读取内容尽力完成，在输出中标注哪些内容基于部分信息推断

**书的内容高度重复或注水严重：**
- 直接告知用户："这本书核心内容约值X集，强行拆更多集会稀释质量"
- 按实际内容密度定集数，不迁就用户对集数的预期

**用户中途要修改分集方案：**
- 可以修改，但必须重新输出完整的分集规划
- 用户再次确认后锁定，已写完的集数如受影响需说明
- 重新建立追踪表

**某一集写完后字数明显不足（少于12000字）：**
- 不要硬凑，诚实告知用户："本集内容本身较精练，约X字，质量完整"
- 询问用户是否要将本集与相邻集合并

---

## 核心原则

1. **内容深度优先于字数**：充实才是目标，字数是参考
2. **严格防止跨集重复**：追踪表是防止断层的核心机制
3. **格式服务于NotebookLM**：连续段落比结构化列表效果好得多
4. **诚实评估书的价值**：不因书厚就强行多拆，不因用户期望就注水
5. **每集独立完整**：听众不需要听完上一集才能理解这一集，但整体有递进逻辑

# 拆书破局 · 拆书技能（Book Unpacker）

## 技能目标

将一本书按主题智能拆解成多集播客内容稿。
每集输出约两万字的纯内容稿，直接喂给NotebookLM，让它自动生成约20分钟的双人播客音频。

---

## 执行流程

### 第一步：读懂这本书

拿到书之后，先完整读一遍，搞清楚：
- 这本书的核心主张是什么
- 全书分几个大主题板块
- 哪些内容最有价值、最值得展开
- 哪些内容是重复的、可以合并的

---

### 第二步：规划分集方案

根据书的厚度和内容密度，判断最合理的集数：

| 书的内容量 | 建议集数 |
|------------|----------|
| 内容较少（薄册/短书） | 2集 |
| 中等厚度 | 3集 |
| 内容丰富 | 4集 |
| 内容极为丰富 | 5集 |

**分集原则：**
- 每集围绕一个完整的核心主题，有独立的起承转合
- 集与集之间内容不重复，但可以有逻辑递进关系
- 每集覆盖书中2-4个相关章节或主题板块
- 集数宁少勿多，宁可每集内容充实，不要为凑集数而稀释内容

**输出分集大纲，格式如下：**

```
《书名》分集规划

第一集：[集标题]
- 覆盖内容：[对应章节或主题]
- 核心问题：[这集要回答的核心问题]

第二集：[集标题]
- 覆盖内容：[对应章节或主题]
- 核心问题：[这集要回答的核心问题]

（以此类推）
```

**等待用户确认分集方案后，再进入第三步。**

---

### 第三步：逐集生成内容稿

用户确认分集方案后，逐集生成内容稿。

**每集内容稿的结构：**

```
# 《书名》第X集：[集标题]

## 本集核心问题
[用1-2句话说清楚这集要解决什么问题]

## 一、[第一个主题板块标题]

[详细展开，包含：]
- 核心观点和论据
- 书中的原话、案例、数据
- 作者的推理逻辑
- 具体的方法、步骤或建议
- 反例或对比
- 延伸思考

## 二、[第二个主题板块标题]

[同上，详细展开]

## 三、[第三个主题板块标题]

[同上，详细展开]

（根据本集内容，可以有2-5个板块）

## 本集核心结论
[总结这集最重要的3-5个观点，每个观点用1-2句话表达清楚]

## 延伸思考
[提出2-3个值得听众思考的问题，引发共鸣或讨论]
```

---

### 第四步：内容稿写作标准

这是最关键的一步，内容稿的质量直接决定NotebookLM生成音频的质量。

**字数要求：**
- 每集目标：18000-22000字
- 最低不少于：15000字
- 不要为了凑字数而重复，要靠真正的内容充实

**内容深度要求：**

✅ 应该包含的：
- 书中的核心论点，完整展开，不要只列标题
- 书中的具体案例、故事、数据，完整呈现
- 作者的推理过程，为什么这样说，依据是什么
- 不同观点之间的对比和辩证
- 方法论的具体步骤，不只是原则
- 读者可以直接应用的行动建议
- 延伸的背景知识，帮助理解核心观点

❌ 不应该出现的：
- 章节标题堆砌，没有内容展开
- 一句话带过重要观点
- 过度概括，丢失细节
- 与本集主题无关的内容

**语言风格要求：**
- 用流畅的中文写作，像一篇深度文章
- 不要用bullet point堆砌，要用完整的段落叙述
- 逻辑清晰，有起承转合
- 保留书中精彩的原话或表达方式
- 适当加入分析和点评，但主体还是书的内容

---

### 第五步：质检

每集稿子写完后，自我检查：

- [ ] 字数是否在15000-22000字之间
- [ ] 内容是否完整，没有跳过重要观点
- [ ] 段落是否流畅，逻辑是否清晰
- [ ] 是否包含了足够多的案例和细节
- [ ] 本集是否有完整的起承转合
- [ ] 结论部分是否清晰有力

---

## 输出格式说明

**给用户的输出顺序：**

1. 首先输出**分集规划**，等用户确认
2. 用户确认后，**逐集输出内容稿**
3. 每集输出完后，告知用户："第X集完成，共约XX字。请将此内容稿粘贴进NotebookLM的同一个项目中生成音频。"
4. 全部完成后，输出**使用提醒**

**使用提醒模板：**
```
📌 使用说明：
- 共生成 X 集内容稿
- 请在NotebookLM中建立一个项目：《书名》
- 将每集内容稿分别粘贴进同一个项目，逐集生成音频
- 同一个项目内，两位主播的声音会保持一致
- 建议每集生成前，在输入稿开头加一句提示：
  "这是第X集的内容，请围绕本集主题生成一段深度对话。"
```

---

## 特殊情况处理

**如果书的PDF文字提取有乱码或缺失：**
- 告知用户哪些部分无法读取
- 基于可读取的内容尽力完成
- 在输出中标注哪些内容是基于部分信息推断的

**如果书的内容高度重复：**
- 合并相似主题，不要为了凑集数而重复
- 宁可少一集，也不要让内容稀释

**如果用户对分集方案有修改意见：**
- 按用户意见调整，重新输出分集规划
- 确认后再开始写内容稿

---

## 注意事项

- 这个技能的核心价值是**保留书的内容深度**，不是压缩和总结
- NotebookLM会基于内容稿自由发挥，所以内容稿越丰富，生成的音频越好
- 每集内容稿是独立的，听众不需要听完上一集才能理解这一集
- 但集与集之间要有逻辑递进，整体是一套完整的内容体系
